# demo

## 1、dependencies 中插件的作用

```text
@expo/vector-icons：提供React Native应用程序中使用的矢量图标库。包括常用的图标集如FontAwesome、Ionicons和MaterialIcons。

@react-navigation/native：React Navigation的核心包，为React Native应用程序提供导航功能。它帮助管理屏幕切换、导航历史和深度链接。

expo：Expo框架的核心，用于快速构建、部署和迭代iOS、Android和Web应用程序。提供了一套工具和服务以简化开发过程。

expo-font：提供在Expo应用中加载自定义字体的方法。帮助在React Native应用程序中添加和使用不同的字体系列。

expo-linking：管理Expo应用中的深度链接。允许你以一种在Web和本地应用中都适用的方式打开链接和处理URL。

expo-router：使用文件系统路由简化Expo应用中的导航，类似于Web开发中的Next.js框架。

expo-splash-screen：管理应用程序的启动屏幕。允许自定义应用加载时显示的初始屏幕。

expo-status-bar：为Expo应用提供可定制的状态栏组件，允许你修改状态栏的外观和行为。

expo-system-ui：允许控制本地UI组件，如状态栏、启动屏幕和其他系统UI元素。

expo-web-browser：提供在应用内打开Web浏览器的功能，允许用户在不离开应用的情况下查看Web内容。

react：用于构建用户界面的React库核心。用于构建组件并管理应用程序的状态和生命周期。

react-dom：为Web应用程序提供React功能，使React能够在浏览器中渲染组件。

react-native：使用React构建移动应用程序的核心库。提供开发移动应用程序所需的基本组件和API。

react-native-reanimated：用于在React Native应用程序中创建平滑动画的库。相比内置的Animated API，它提供了更好的性能和对动画的控制。

react-native-safe-area-context：处理React Native应用程序的安全区域内边距，确保内容在带有缺口、圆角和其他非矩形屏幕形状的设备上正确显示。

react-native-screens：通过使用本地导航屏幕原语优化导航性能，改进内存使用和过渡效果。

react-native-web：使React Native组件和API可以在Web上使用，从而可以用相同的代码构建跨平台应用程序。
```
